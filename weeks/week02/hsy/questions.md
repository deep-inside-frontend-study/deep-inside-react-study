### 1. 양방향 바인딩은 상태 흐름을 비명시적으로 만든다?

양방향 바인딩은 UI 변경이 상태 변경으로 이어짐, 이로 인해 크게 아래 3가지 문제가 있을 수 있다고 봄

1. View가 상태를 직접 변경 ⇒ 뷰와 비즈니스 로직 사이의 경계가 흐려짐
2. 상태 변경 로직 분산 ⇒ 상태 변경 흐름이 불명확, 예측 가능성 낮음
3. 상태 변경 시점을 프레임워크가 암묵적으로 처리 ⇒ 개발자가 조절하기 어려움

<br />

### 2. React가 단방향 바인딩으로 구현된 이유?

- React는 **예측 가능성**을 위해 단방향 바인딩을 채택함 → 상태 변경은 항상 명시적인 이벤트와 함수 호출을 통해서만 발생
- React가 선언적 UI 모델을 채택한 것이 비슷한 맥락이라고 봄
  - 선언적 UI: `UI = f(state)`
  - UI는 상태의 결과물이며, 상태가 바뀌면 UI가 다시 계산된다
- 상태는 단방향으로 명시적인 이벤트와 함수 호출만으로 변경됨 → 예측 가능성이 높아지고 복잡한 상태를 안정적으로 관리 가능
