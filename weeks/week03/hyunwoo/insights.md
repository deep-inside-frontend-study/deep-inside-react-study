## 이번 주 핵심 요약

### 7장

- 다양한 컴파일러에 대해 알게 되었습니다. Babel, SWC, ESBuild 등과 같은 다양한 툴이 존재합니다. 책에는 컴파일러의 속도 측면이 부각되었지만 실제로 컴파일의 결과도 굉장히 중요한 요소라고 합니다. 즉, ESBuild가 Vite에서 개발 모드에서 실행되는 것은, 정교하게 트랜스파일링되지 못하더라도 빠른 속도를 보장하기 때문입니다. 반면, 프로덕션 빌드 시에는 정교한 결과물이 필요한 경우가 많습니다. 개인적으로 시간을 내서 책에서 다루지 않은 트랜스파일러들의 차이점을 공부해야겠다는 생각을 했습니다.
- React Fragment에 대한 개념을 다시 한번 공부했습니다. 사실 동료 분들의 코드를 본 경우 굳이 필요하지 않은 <div> 태그 등으로 감싼 경우가 종종 보입니다. 고작해야 <div> 하나 더 감싼거 아냐? 라고 생각하겠지만, 변환된 결과물 자체가 달라지기도 하고, 무엇보다 DOM Tree에서 depth 자체가 달라지다 보니 스타일 자체도 달라질 수 있습니다. 관성적으로 <div> 등으로 감싸려는 습관도 지양하면 좋겠다고 느꼈습니다.

### 8장

- React의 재조정 과정에 대해 알 수 있었습니다. 사실, React 재조정 과정은 공식문서에도 찾아보기 어려워 보입니다. 대략 Diffing 알고리즘 정도가 나와있습니다. (오히려 이 부분은 이전 공식문서가 좀 더 잘 나온 것 같네요)
- 그런 의미에서 Fiber 내부 로직을 공부할 수 있게 되어 좋았다고 생각합니다. 다만 조금 더 깊이있게 공부해봐도 좋을 부분이긴 합니다. 예를 들면, 왜 연결리스트 구조를 선택했는지? 기존엔 어떤 구조였는지? 추가적으로 이와 연계된 (React18에서 추가된) useTransition, useDefferedValue 등과 같이 연계해서 공부하면 좋을 것 같습니다.
- 추가로, 커밋 과정에 대한 설명이 없는 것은 조금 아쉬웠습니다. 파이버 노드에서 계산된 결과물이 실제 DOM에 붙는 과정이 나왔다면 좀 더 이해가 쉽지 않았을까 하는 생각이 듭니다.
- React에서 채택한 얕은 비교 방식이 어떤식으로 리렌더링을 막을 수 있는지 알 수 있었습니다. 추가로 렌더와 커밋 부분을 명확하게 이해하는 것이 중요하다고 생각합니다. 우리가 사용하는 useMemo, useCallback, memo 함수가 어떤 과정에서 작동하는 지에 대해 이해하는 것이 중요하다고 생각합니다.

### 9장

- 평소 Context API를 자주 쓰지는 않아, 정확히 어떤 로직으로 리렌더링되는지 알지 못했습니다. 내가 특정 Context에서 특정 상태만 사용하더라도 다른 트리거에 의해 리렌더링 될 수 있다는 점에 대해 알게 되었습니다. 개인적인 생각으론, Zustand나 @tanstack/react-query의 select와 같은 구문이 이런 상황에서의 불필요한 리렌더링을 막아주는 방향으로 진화했다고 들었는데, 이 부분을 추가로 공부해야겠습니다.
- [컴포넌트와 Hook은 순수해야 한다는 원칙](https://react.dev/reference/rules/components-and-hooks-must-be-pure)에 대해 알게 되었습니다. 이 부분은 리액트 공식 문서에 잘 나와있네요.
