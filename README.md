# Inside React Study Group
> ‘다시 깊게 익히는 인사이드 리액트’ 함께 읽고 토론하는 스터디 저장소

## 스터디 안내

이 저장소는 아래 목적으로 운영됩니다:

✔ React의 내부 작동 원리를 **구조 중심으로 이해**  
✔ 책 내용을 **단순 요약이 아닌 재해석/Explain으로 정리**  
✔ 실습 코드 / 의견 / 질문을 모두 함께 공유  

## 스터디 목표
- React의 **렌더링 메커니즘**, **상태 관리**, **Fiber 구조**를 내부 동작 관점에서 이해
- “어떻게 쓰는가”가 아니라 “왜 이렇게 동작하는가”를 설명할 수 있는 수준 도달
- 실무 코드에 적용 가능한 인사이트 정리


## 스터디 진행 방식

1️⃣ 매주 **각자 1~3주 분량**을 정리해요.

2️⃣ 각자 정리한 내용을 **Summary + Q&A + Opinion** 형식으로 작성해요.

3️⃣ 스터디 당일 **뽑기**로 대표 2명이 설명해요.

4️⃣ 서로 질문하고 설명해주며 함께 이해 확장, 단순 요약 ❌ → **이해한 방식으로 재해석 ⭕**

## weeks/weekXX/ 파일 정의

📄 summary.md

챕터 핵심 요약
- 해당 주차에 읽은 챕터의 핵심 내용 정리
- 책의 흐름을 따라가되, 중요 개념 위주로 압축
- 인용보다는 자기 언어로 재정리하는 것을 권장

포함하면 좋은 내용
- 챕터의 주제와 목적
- 핵심 개념 / 키워드
- 중요한 예시나 비유
- 한 줄 요약 (TL;DR)

❓ questions.md

토론 및 이해를 위한 질문 모음
- 스터디 중 토론·토의가 가능한 질문
- 읽으면서 이해가 가지 않았던 부분에 대한 질문
- 정답이 하나가 아닌, 생각을 나눌 수 있는 질문을 우선

질문 유형 예시
- “왜 이런 설계를 했을까?”
- “이 개념은 어떤 상황에서 문제가 될 수 있을까?”
- “A 방식과 B 방식의 트레이드오프는?”
- “이 내용이 실제 프로젝트에서 어떻게 쓰일까?”

> 질문은 ‘모르겠다’에서 끝나지 않고, 대화의 출발점이 되는 것을 목표로 한다.


💡 insights.md

확장 학습 & 개인 인사이트

챕터 내용을 기반으로 한 딥다이브와 경험 정리

포함 가능한 내용

- 딥다이브 자료(공식 문서 (Docs), 참고 영상, 발표 자료, 아티클 등 등)
- 직접 해본 것(테스트해본 코드, 간단한 실험 결과)
- 경험 기반 인사이트 공유
    - 실무에서 비슷한 문제를 어떻게 해결했는지
    - 과거에 했던 선택의 장단점
    - 지금 다시 한다면 어떻게 개선할지
